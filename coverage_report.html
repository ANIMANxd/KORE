Critical Coverage Gaps

run.py — 0% Coverage

All 620 statements untested. Every CLI command (setup, ingest, extract, review, status, slack-setup, github-setup, notion-setup, jira-setup, check-expiry, reextract, export-skills, reset-db, tui) is completely untested. This is by far the biggest gap in the codebase.

ingestion/loader.py — 58% Coverage

Interactive setup flows (Rich console prompts, Progress bars, _click.prompt calls) are mocked out in tests. The actual live API paths inside Progress contexts aren't hit. The loading logic itself is tested, but the UI-heavy paths and error handling within those flows are not.

ingestion/chunker.py — 63% Coverage

Some chunker branches untested: Zendesk article chunking, Google Drive document chunking, Confluence table handling, DocumentChunker semantic split for long PDF pages (>4000 chars), and some edge cases in generic fallback.

providers/llm.py — 57% Coverage

complete_json() repair loop partially tested via test_json_repair.py, but the actual litellm.completion() call paths, streaming responses, and error recovery are not covered.

providers/embedder.py — 40% Coverage

embed_text() and embed_batch() call LiteLLM — mocked in backend tests but the retry logic, batching edge cases, and failure paths aren't fully covered.

engine/graph.py — 54% Coverage

LangGraph pipeline wiring and run_extraction() runner. Individual nodes are tested, but the graph wiring and state transitions between nodes are not tested end-to-end.

engine/nodes/formatter.py — 19% Coverage

Rule formatting node — turns extraction state into YAML-ready output. Completely untested in isolation from the graph.

engine/nodes/retriever.py — 22% Coverage

Vector search retriever — queries the vector store. Not tested end-to-end because it requires a live vector store and embeddings.

output/writer.py — 18% Coverage

YAML save/load/update logic. Only output/schema.py (Pydantic models) is tested — the actual file I/O and directory creation paths are not.
